一：
webpack的构建流程：

1.entry入口

2.递归解析entry依赖的所有的module，根据module.rules的loader进行转换

3.解析当前module依赖的module，这些module以entry为单位进行分组，即为一个chunk

4.chunk转换成文件输出output

5.plugin特定时机注入扩展逻辑，用来优化

webpack的打包过程

1.安装webpack、webpack-cli

2.安装各种loader

3.配置webpack.config.js

4.安装webpack-dev-server

二：
loader和plugin的不同

loader用于对模块的源代码转换 ，是对打包前的模块预处理。

plugin目的解决loader无法实现的事情，是打包过程中基于事件机制的优化

如何开发loader

1.确定单一的loader的功能

2.链式调用

3.遵循webpack的规则和结构，输入输出都是字符串，独立即用

如何开发plugin

1.供实例设置options的构造函数

2.构造函数的原型prototype定义apply方法

3.apply方法注入compiler对象，挂载webpack事件钩子

4.事件钩子回调函数中的compilation对象处理webpack内部实例的特定数据

5.事件钩子回调函数中的回调函数调用

webpack实现vue项目打包任务

1.npm install webpack webpack-cli webpack-dev-server --save-dev

2.npm install style-loader css-loader less-loader 打包样式文件

3.npm install url-loader 打包图片文件

4.npm install vue-loader 打包.vue文件

5.npm install babel-loader @babel/core @babel/preset-env 编译文件

6.npm install clean-webpack-plugin html-webpack-plugin copy-webpack-plugin 插件处理自动清除，html文件，复制静态文件

7.webpack.common.js 配置webpack

8.npm run webpack 打包

9.npm run webpack-dev-server 打开项目
